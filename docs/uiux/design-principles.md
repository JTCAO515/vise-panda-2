# 设计原则

> VisePanda 的核心设计哲学

## 1. 管家心态，非工具心态

VisePanda 不是"用户自己操作的旅行规划工具"，而是一个**懂中国的私人管家**。

- 界面应该像管家在帮你办事，而不是让你自己填表
- 信息呈现要预判用户下一步需要什么
- 减少选择，增加建议

## 2. 外国人优先

所有设计从"不会中文的外国游客"视角出发。

- 文字清晰简单，避免旅行行业黑话
- 关键信息要显眼（签证要求、支付方式、紧急联系方式）
- 翻译功能必须是原生体验，不是跳转第三方

## 3. 移动为先

手机是旅行者在路上的主要设备。

- 单手操作优先
- 底部导航 + 拇指热区
- 减少打字，多用点选和语音
- 离线能力优先

## 4. 冷静、可信、专业

中国旅行对很多人来说是高焦虑决策。界面要传递信任感。

- 冷静的蓝色调 + 温暖橙色作点缀
- 信息密度适中，不堆砌
- 数据来源清晰标注
- 错误提示要有解决方案，不是只报错

## 5. 渐进式交付

不一次把所有功能扔给用户。

- 行前：规划 + 咨询为主
- 行程确认后：路上服务逐步解锁
- 界面随用户状态变化，而非一成不变
